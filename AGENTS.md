## 项目定位

本项目是“商用密码应用安全性评估实施工具”的 Windows 本地桌面实现。目标是依据当前需求文档记录的业务流程和现有行为协议，开发一套本地单机工具，支撑项目管理、基本信息、四类技术域测评、量化评估、风险判定、密码产品、证据图片、知识库、评分、Excel 导入导出、问题清单、数据兼容和安装运行。

本项目当前的核心交付不是重新发明业务规则，而是把需求文档中已经明确的当前行为稳定、可测试、可维护地实现出来。

## 架构原则

- UI 层只负责界面展示、控件状态、信号连接和用户反馈。
- 应用服务层负责业务用例、事务边界和跨模块协调。
- 领域层集中维护枚举、评分公式、量化规则、关联矩阵、问题清单模板和映射常量。
- 仓储层负责 ORM 模型、Repository、查询和会话生命周期。
- 适配器层负责 Excel、证据文件、缩略图、配置、日志、资源路径和打包。
- 不要把保存、评分、导入导出、证据文件复制等业务逻辑写进窗口类。
- 不要在多个页面分散维护 Excel 列协议、评分规则、测评单元映射或关联矩阵。

## 关键业务不变量

以下规则不得随意改变：

- 当前需求文档记录的是验收基线，包含用户可见行为和静默行为。
- 详情页只读加载不得创建、修改、删除、提交或回滚数据。
- 评分页技术域刷新按兼容要求可以补建缺失的空量化评估记录；这是一条有副作用的兼容行为，必须被测试覆盖。
- 主窗口手动保存应先保存当前详情，再保存基本信息。
- 从基本信息页切换到非基本信息页时，只静默保存基本信息页，不主动保存技术域详情。
- 自动保存默认静默，不弹成功提示；失败至少记录日志。
- 量化评估和证据记录通常关联测评单元详情。
- 密码产品存在历史兼容语义：详情记录和外层对象记录并存，项目范围解析必须同时承认两类关联。
- 软删除只写回收站记录，不删除项目业务数据。
- 硬删除服务不得删除磁盘上的证据图片文件。
- 证据记录不保存绝对路径，只保存文件名、排序和业务关联。
- 证据文件和数据库记录允许非强一致，文件缺失时保留列表文本并容错。
- 管理域评分修改后会自动落库并刷新评分汇总。
- 指标 8、12、17 始终不适用。
- 评分公式、层面权重归一、总分 70/30 分配、符合统计必须按需求文档执行。
- Excel sheet 名称、列顺序、合并单元格、颜色、列宽、空值表达和产品文本格式属于验收范围。
- 问题清单只处理四个技术域，不包含管理域问题。
- 下拉框悬停滚轮不得误切换选项。
- 临时提示、日历弹层、下拉弹层等辅助窗口必须绑定父窗口，避免出现标题为 `python` 的临时窗口。

## 数据与迁移约束

- 本地数据默认存 SQLite 文件。
- 安装包运行后数据文件应位于当前用户可读写的应用数据目录，不应写入安装目录。
- 迁移必须幂等，可重复运行。
- 迁移失败应记录警告并尽量保持旧库可打开。
- 兼容旧枚举和历史字符串时，不得因非标准值导致详情页崩溃。
- 关联完整性检查必须只读，不能自动删除、迁移或修复记录。

## UI 与交互约束

- 进行 UI 设计、界面重设计、原型、设计评审、从截图/Figma/图片/URL 实现界面，或任何明显涉及产品视觉与交互方案的工作时，必须先使用 `product-design:index` 技能进行路由。
- 如果是新界面、原型、重设计或 UI build，按 Product Design 流程先走 `get-context`；没有 URL、截图、Figma、mockup、源图或现有代码视觉目标时，不得直接开工实现。
- 对没有明确视觉目标的新 UI 工作，必须先完成设计 brief，再通过 Product Design 的 ideation 流程给出视觉方案，并等待用户选择后再进入实现。
- 桌面端以 PySide6 为主。
- 使用统一 QSS、页头、分组、表单行、分区标题和紧凑布局。
- 保存按钮应位于详情页页头右上角或操作区，不要放在底部占满整行。
- 物理、设备、网络、应用页面采用左侧对象列表 + 右侧详情的主从布局。
- 左侧列表不要明显挤压右侧详情；长名称要换行。
- 表格下拉框要垂直居中，不得被单元格裁切。
- 宽表格优先内部滚动，不要造成整个窗口不必要横向滚动。
- 日期控件要支持清空、手输提交、弹层选择和最小最大日期约束。

## 测试要求

优先按测试驱动实现高风险规则：

- D/A/K/Ra/Rk 联动和幂等保存。
- 对象得分公式、层面得分、总分和符合统计。
- 风险缓释后的最终风险等级。
- 密码产品同证书同步和历史关联兼容。
- 证据目录清洗、导入、删除、重编号。
- 项目软删除、恢复、硬删除范围。
- 子系统同步网络和应用页且不清空已有数据。
- Excel 导入回滚和错误上下文。
- Excel 导出格式结构。
- 问题清单模板、风险颜色和合并规则。
- 评分页刷新补建空量化评估记录。
- UI 主窗口标题、下拉框滚轮保护、管理域下拉框不裁切。

最终验证命令：

```powershell
python -m pytest -q
python -m ruff check .
powershell -ExecutionPolicy Bypass -File scripts/check_resources.ps1
```
